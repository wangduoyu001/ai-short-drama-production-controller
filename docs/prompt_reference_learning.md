# 提示词参考语料学习说明

## 目的

用户提供的提示词合集只作为学习样本，用来理解一套成熟工作流如何组织输入、输出、字段、分镜、资产和生成提示词。

它不作为本项目的名称、默认模板或固定题材。

## 学习重点

1. 原文必须被完整追踪，不能只看摘要。
2. 角色、场景、道具要先建库，再进入分镜和生成提示词。
3. 资产图、生图、首帧、尾帧、生视频、负面提示词、备用镜头要分开。
4. 单镜推理要读取当前镜头、前后镜头、角色库、场景库、道具库。
5. 批量推理要保持镜头序号、字段数量和回填关系稳定。
6. 多人物对白要明确说话人、闭口人、空间锚点和嘴型状态。
7. 宫格方案只用于高风险镜头，不是所有镜头的默认方案。
8. 生成提示词要保持字段密度，不能压成泛化短句。

## 不学习的内容

1. 不学习来源名称作为项目名。
2. 不复制样例人物、样例场景、样例道具。
3. 不把样例风格写成全局默认风格。
4. 不把参考提示词正文压缩后塞进代码。
5. 不用参考语料替代用户当前输入。

## 转成项目能力

这批语料应转成以下能力：

```text
source_segments 原文切片
source_coverage 原文覆盖
character_cards 角色卡
scene_plan 场景计划
asset_lock 资产锁定
shot_inference 单镜推理
batch_inference 批量推理
grid_strategy 宫格策略
prompt_density_qa 提示词密度质检
```

核心原则：学工艺，不抄外壳。
