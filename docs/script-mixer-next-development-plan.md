# 文案驱动本地混剪：后续开发计划

## 一、已经锁定的产品边界

### 1. 源视频只处理前40秒

默认配置：

```json
{
  "media_scan": {
    "maximum_source_process_seconds": 40.0
  }
}
```

定义：

```text
源视频可处理区间 = 0秒 ～ min(原视频实际时长, 40秒)
```

超过40秒的尾部：

- 不参与场景检测。
- 不生成镜头区间。
- 不生成缩略图。
- 不进入视觉模型分析。
- 不构建语义向量。
- 不进入候选检索。
- 不允许时间线引用。

仍然保留：

- 原视频实际总时长 `duration`。
- 实际入库时长 `indexed_duration`。
- 被忽略尾部 `ignored_tail_seconds`。

该限制针对每个原始素材，不限制最终成片总时长。最终视频仍可由多个素材的前40秒片段组合而成。

### 2. 本地优先

- 原素材只读。
- 不自动上传素材。
- 不自动发布视频。
- 不自动下载大模型。
- 没有本地模型时必须明确降级。
- 所有降级原因必须写入报告。

### 3. Codex的角色

Codex负责：

- 开发和维护系统。
- 修改规则与模板。
- 排查失败报告。
- 扩展新模块。

日常剪辑由本地Python、FFmpeg、SQLite、Whisper和Ollama执行，不应让Codex逐条参与普通视频生产。

已经具备：

- 单素材前40秒处理上限。

- 文案画面意图。
- 多来源镜头编排。
- 真实配音、原声、混合和静音模式。

# 三、开发阶段与优先级

## P0：40秒处理窗口验收

状态：当前开发分支已实现，等待完整CI和真实电脑验证。

### 开发任务

1. 配置增加 `maximum_source_process_seconds=40.0`。
2. FFmpeg场景检测命令增加40秒终止参数。
3. 切镜区间使用 `indexed_duration`，不使用原始总时长。
4. 缩略图时间点必须小于等于40秒。
5. 视觉分析和向量构建只读取已入库镜头。
6. 数据库增加：
   - `indexed_duration`
   - `ignored_tail_seconds`
7. 旧数据库自动迁移。
8. 原先索引超过40秒的旧记录自动重建，不要求删库。
9. `catalog-status` 显示原始时长、入库时长和忽略尾部。

### 验收标准

- 95秒素材只生成 `0～40秒` 的镜头。
- 所有 `source_end <= 40.0`。
- 场景检测FFmpeg命令包含40秒限制。
- 视觉分析队列中不存在40秒后的缩略图。
- 修改配置后旧索引自动升级。
- 40秒以内素材不受额外裁剪。
- 快速扫描和完整扫描行为一致。

---

## P1：真实Windows电脑集成验收

优先级：最高。

目的：确认系统不是只在GitHub Actions里活得很好。

### 开发任务

1. 新增 `integration-check` 命令。
2. 自动检查：
   - FFmpeg和FFprobe版本。
   - NVIDIA驱动和显卡可见性。
   - Ollama服务。
   - Whisper CLI和权重。
   - 字体和libass字幕支持。
   - 素材目录读权限。
   - 输出目录写权限。
3. 新增真实试剪清单：
   - 横屏素材。
   - 竖屏素材。
   - 有原声素材。
   - 无原声素材。
   - 10秒以内素材。
   - 超过40秒素材。
   - 中文真实配音。
4. 增加本机性能记录：
   - 扫描耗时。
   - 场景检测耗时。
   - 单镜头视觉分析耗时。
   - 向量构建速度。
   - Whisper转写耗时。
   - FFmpeg渲染倍率。
   - GPU显存峰值。
5. 输出 `integration_report.json`。
6. 对失败项给出明确修复建议，不只返回异常堆栈。

### 验收标准

- 可以从空数据库完成一次完整入库。
- 可以生成一条真实30～40秒预览视频。
- 输出视频可播放且音画同步。
- 长素材没有读取40秒后的画面。
- 字幕中文字体正常。
- 失败后重复执行可以续跑，不重做全部步骤。

## P2：可返修时间线

优先级：高。

目的：自动粗剪一定会有不合适镜头。没有返修能力的一键成片，只是一键把错误固化。

### 开发任务

1. 给每个时间线片段增加：
   - `locked`
   - `review_status`
   - `replacement_reason`
   - `candidate_rank`
2. 增加命令：

```text
review-project
replace-segment
lock-segment
unlock-segment
rerender-project
```

3. 支持按以下条件替换镜头：
   - 指定片段编号。
   - 排除某个素材来源。
   - 排除某个镜头。
   - 指定画面关键词。
   - 指定景别。
   - 指定是否保留原声。
4. 已锁定片段不得被重新规划覆盖。
5. 只重新渲染受影响部分，避免整条视频重做。
6. 保存每次返修前后的差异记录。

### 验收标准

- 可以替换单个错误镜头。
- 替换后其他片段时间不漂移。
- 已锁定片段在重新规划后保持不变。
- 可以回退到上一个时间线版本。
- 不生成 `final_v2`、`最终版3` 等垃圾文件名。

---

## P3：本地背景音乐和音效库

优先级：高。

### 新模块

```text
script_mixer/audio_library.py
script_mixer/music_selector.py
script_mixer/sound_effects.py
```

### 数据结构

背景音乐记录：

```text
track_id
source_path
duration
bpm
energy
mood
genre
loopable
license_status
usable
```

音效记录：

```text
effect_id
source_path
duration
category
intensity
keywords
license_status
usable
```

### 开发任务

1. 扫描本地音乐和音效目录。
2. 识别时长、响度、静音段和基础节奏。
3. 人工标签与本地模型标签并存。
4. 根据文案情绪曲线选择音乐。
5. 支持音乐循环、裁切、淡入和淡出。
6. 配音出现时自动压低音乐。
7. 转折、数字强调、镜头切换和结尾支持音效事件。
8. 音效默认少用，避免每个切镜都像廉价综艺。
9. 所有音乐和音效必须记录授权状态。
10. 未确认授权的素材默认阻止正式导出。

### 验收标准

- 可以只加背景音乐。
- 可以只加音效。
- 可以同时混合配音、原声、音乐和音效。
- 配音清晰度不被音乐覆盖。
- 最终峰值不过载。
- 音频计划完整写入 `timeline.json` 和 `report.json`。

## P4：剪辑风格模板

优先级：高。

目的：同一套规则不可能同时适合知识口播、商业广告、历史故事和AI短剧。人类已经证明，统一模板通常只会统一地难看。

### 预置模板

1. `information_flow`：信息流口播。
2. `business_ad`：商业广告。
3. `knowledge`：知识解释。
4. `storytelling`：故事叙述。
5. `short_drama_recap`：短剧解说。

### 模板字段

```text
shot_duration_range
source_count
source_reuse_gap
transition_density
literal_vs_metaphor_ratio
preferred_shots
subtitle_style
audio_mode
music_energy
hook_pacing
ending_hold
```

### 开发任务

1. 把当前硬编码默认值迁移到模板。
2. CLI增加 `--profile`。
3. 项目生成时复制模板快照，避免以后模板变化导致旧项目无法复现。
4. 支持用户本地自定义模板。
5. 模板必须通过JSON Schema校验。

### 验收标准

- 同一文案使用不同模板能产生明显不同但合法的时间线。
- 模板不覆盖用户显式参数。
- 项目可完全复现。

---

## P5：提升画面匹配准确率

优先级：中高。

### 开发任务

1. 单镜头从一张中点图升级为多帧采样：
   - 25%位置。
   - 50%位置。
   - 75%位置。
2. 多帧合并描述，减少中点恰好模糊或被遮挡的问题。
3. 增加近重复镜头检测：
   - 感知哈希。
   - 颜色直方图。
   - 向量相似度。
4. 增加OCR和字幕水印检测。
5. 增加人物、主体和场景连续性评分。
6. 增加运动强度评分。
7. 增加画面安全区判断，避免字幕遮挡主体。
8. 所有采样仍严格限制在每个源视频前40秒。

### 验收标准

- 不连续重复使用近似画面。
- 明显水印素材默认不可用。
- 主体被字幕遮挡的概率下降。
- 候选报告能解释匹配和排除原因。

---

## P6：TTS适配层

优先级：中。

真实配音优先，TTS只是备用入口。

### 开发任务

1. 定义统一接口：

```text
synthesize(text, voice, speed, emotion, output_path)
```

2. 支持本地TTS适配器。
3. 外部服务适配器必须显式配置，不默认上传文案。
4. 生成音频后复用现有Whisper对齐流程。
5. 记录：
   - 引擎。
   - 音色。
   - 语速。
   - 情绪参数。
   - 生成时间。
6. 支持单句重新生成，不要求整条配音重做。

### 验收标准

- TTS输出可以直接进入现有时间线。
- 单句替换后字幕和镜头局部更新时间。
- 未配置TTS时不影响真实配音流程。

---

## P7：人工审片页面

优先级：中。

### 页面能力

- 视频预览。
- 时间线片段列表。
- 文案与字幕。
- 当前镜头和候选镜头。
- 原素材来源。
- 匹配分数和原因。
- 锁定、替换、禁用和重渲染。
- 质量警告。

### 开发任务

1. 第一版使用本地HTML或轻量Web页面。
2. 不先做复杂画布编辑器。
3. 页面只调用稳定服务层，不直接拼FFmpeg命令。
4. 所有人工修改写回项目JSON。

### 验收标准

- 不打开代码也能完成一次单镜头返修。
- 页面刷新后修改不丢失。
- 不允许页面绕过QA直接正式导出。

## P8：导演系统API接入

优先级：中。

### API

```text
POST /script-mixer/scan
POST /script-mixer/enrich
POST /script-mixer/embed
POST /script-mixer/plan
POST /script-mixer/render
GET  /script-mixer/projects/{id}
POST /script-mixer/projects/{id}/segments/{segment_id}/replace
POST /script-mixer/projects/{id}/segments/{segment_id}/lock
```

### 开发任务

1. 使用FastAPI封装现有服务层。
2. 长任务使用本地任务队列。
3. 提供进度、失败原因和重试入口。
4. 导演画布增加“本地混剪”节点。
5. 节点只传路径引用和项目ID，不传大视频二进制。

### 验收标准

- 导演系统可以创建混剪项目。
- 可以查看任务进度。
- 可以打开预览和返修页面。
- API重启后任务状态可恢复。

## P9：工程文件导出

优先级：中低。

### 顺序

1. FCPXML。
2. 达芬奇可导入时间线。
3. Premiere兼容XML。
4. 剪映只在格式可稳定验证时接入。

### 输出内容

- 原素材路径。
- 源入点和出点。
- 时间线入点和出点。
- 速度变化。
- 音频轨道。
- 字幕。
- 锁定状态。

### 验收标准

- 导入后镜头顺序和时长一致。
- 不复制原素材。
- 丢失素材时有明确报告。

## P10：批量生产和性能治理

优先级：最后执行，不能抢在单条链路稳定之前。

### 开发任务

1. 批量任务队列。
2. GPU任务互斥与限流。
3. FFmpeg并发上限。
4. 失败续跑。
5. 缓存清理策略。
6. 项目冷归档。
7. 数据库备份和恢复。
8. 批量报告汇总。

### 验收标准

- 单条失败不阻塞整个队列。
- 重启后任务可以恢复。
- 不因并发导致显存耗尽。
- 不重复分析已有镜头。

# 四、推荐实施顺序

```text
P0 40秒限制
→ P1 真实电脑集成验收
→ P2 可返修时间线
→ P3 背景音乐和音效库
→ P4 剪辑风格模板
→ P5 匹配准确率
→ P6 TTS适配层
→ P7 人工审片页面
→ P8 导演系统API
→ P9 工程文件导出
→ P10 批量与性能治理
```

不建议现在直接开发复杂导演画布。没有真实电脑验收和返修能力时，画布只是把尚未稳定的后端包上一层更昂贵的外壳。
