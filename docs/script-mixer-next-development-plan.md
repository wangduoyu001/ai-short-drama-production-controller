# 文案驱动本地混剪：当前状态与后续路线

## 一、产品目标

最终工作方式已经锁定：

```text
系统自动完成粗剪
→ 输出预览视频
→ 输出剪映可编辑镜头、原声、配音、字幕和候选
→ 尝试创建剪映草稿
→ 用户在剪映中自由拖动和修改
→ 剪映正式导出
```

系统不以“AI永远不会切错”为前提。目标是减少错误，同时让剩余错误容易修改。

## 二、固定边界

### 每个源视频只处理前40秒

```text
0秒 ～ min(原视频实际时长, 40秒)
```

40秒后的内容不进入：

- 场景检测。
- 镜头区间。
- 缩略图。
- 视觉分析。
- 语义向量。
- 候选检索。
- 时间线。
- 剪映代理和候选包。

数据库分别保存：

```text
duration
indexed_duration
ignored_tail_seconds
```

该限制不限制最终成片总时长。

### 本地优先

- 原素材只读。
- 不自动上传素材。
- 不自动发布。
- 默认不下载模型。
- 缺少模型时明确降级。
- 本机路径只写入本地忽略配置。

### 剪映双保险

```text
标准编辑包：长期稳定底座
剪映草稿：可选自动入口
```

标准编辑包使用普通MP4、WAV、SRT和CSV。剪映草稿格式属于私有格式，版本变化不能让整个流程失效。

# 四、当前必须完成的真实电脑验收

代码层已经准备好，但以下结果不能在GitHub Actions里假装验证。

## Windows环境

准备：

- 剪映专业版。
- 完整FFmpeg。
- Python 3.10以上。
- 至少8个真实视频素材。
- 至少一个超过40秒素材。
- 至少一个有原声素材。
- 至少一个无原声素材。
- 一份中文文案。
- 一段真实中文配音。

运行：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/setup_jianying_windows.ps1

powershell -ExecutionPolicy Bypass -File scripts/run_jianying_project.ps1 `
  -MediaRoot "D:\视频素材" `
  -Script "input.txt" `
  -Voice "voice.wav" `
  -DraftRoot "D:\JianyingPro Drafts"
```

验收：

- `final.mp4`可播放。
- 没有明显卡帧、黑帧和音画漂移。
- 所有源时间码不超过40秒。
- 剪映草稿可见。
- 视频镜头是独立片段。
- 镜头边缘可以向前或向后拖动。
- 原声和配音是独立轨道。
- 字幕可编辑。
- 候选镜头可以拖入替换。
- 草稿失败时标准编辑包仍可手工导入。

真实验收报告：

```text
.runtime/script_mixer/integration_report.json
```

# 五、后续开发优先级

## P4：切点质量与画面稳定性

优先级：最高。

目标：减少需要人工修改的技术性和语义性切点问题。

任务：

1. 单镜头25%、50%、75%多帧采样。
2. 运动模糊和曝光突变检测。
3. 切点前后帧差异评分。
4. 避免动作进行到一半切走。
5. 避免镜头开始和结束的不稳定区。
6. 近重复镜头检测。
7. 水印、字幕和OCR检测。
8. 主体安全区检测。
9. 候选报告解释排除原因。
10. 继续严格限制在源视频前40秒。

验收：

- 明显模糊帧不作为镜头第一帧。
- 连续近重复画面减少。
- 动作中断切点减少。
- 人工拖动切点的次数下降。

## P5：剪辑风格模板

预置：

```text
information_flow
business_ad
knowledge
storytelling
short_drama_recap
```

模板控制：

- 镜头时长。
- 切镜密度。
- 直接画面和隐喻画面比例。
- 景别偏好。
- 来源数量。
- 字幕样式。
- 音频模式。
- 开头和结尾节奏。

每个项目保存模板快照，保证旧项目可复现。

## P6：背景音乐和音效库

任务：

- 扫描本地音乐和音效目录。
- 记录时长、BPM、能量、情绪和授权状态。
- 音乐循环、裁切、淡入淡出。
- 配音时自动压低音乐。
- 转折、强调和结尾音效事件。
- 配音、原声、音乐、音效四轨导出到剪映。
- 未确认授权的音频阻止正式交付。

## P7：TTS适配层

真实配音优先，TTS作为备用。

要求：

- 本地适配器优先。
- 外部服务必须显式配置。
- 固定角色和音色ID。
- 生成完整主配音轨。
- 单句重生成。
- 生成后复用Whisper对齐。
- 单句替换后局部更新时间线和字幕。

## P8：人工审片页面

目标：不打开代码也能完成审核和替换。

页面包括：

- 预览视频。
- 时间线片段。
- 文案和字幕。
- 当前镜头和候选。
- 锁定、替换、禁用和回退。
- QA警告。
- 一键重新导出剪映包。

第一版只做轻量本地页面，不先开发复杂画布。

## P9：导演系统API

使用FastAPI封装：

```text
scan
enrich
embed
plan
review
replace
export-jianying-package
render
```

导演系统只传项目ID和路径引用，不上传大视频二进制。

## P10：其他工程格式

顺序：

```text
FCPXML
→ 达芬奇时间线
→ Premiere XML
→ 其他剪辑软件
```

剪映标准编辑包和可选草稿已经优先完成。

## P11：批量生产与性能治理

最后执行：

- 批量任务队列。
- GPU互斥和限流。
- FFmpeg并发上限。
- 失败续跑。
- 缓存清理。
- 冷归档。
- 数据库备份和恢复。
- 批量质量报告。

单条链路没有在真实电脑稳定前，不开放大规模并发。批量制造错误确实很高效，但这不是目标。
