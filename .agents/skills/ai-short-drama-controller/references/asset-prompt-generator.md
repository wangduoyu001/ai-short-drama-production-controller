# Asset prompt generator / 资产生图提示词生成契约

## Purpose / 用途

本契约只用于：根据已经确认的资产清单，生成完整、统一、可直接使用的资产生图提示词。

不得重新执行资产提取，不得根据剧本擅自新增、删除或改变资产。

- 已确认资产清单决定：`生成什么`
- 剧本与相关剧情决定：`以什么状态、视角、组合和顺序生成`
- 已审核资产图决定：`下游资产必须继承什么`

## Input gate / 输入完整性闸门

生成提示词前必须检查现有资料是否足以准确判断：

- 人物年龄、身份、外貌、体型、发型、服装和长期变化
- 场景时代、规模、布局、时间、天气和光线
- 核心载体的结构、用途和关键剧情状态
- 资产之间的依赖关系与生成顺序
- 多人物组合、站位、运动和使用关系
- 项目统一风格、色调、材质和画面语言

仅凭资产清单不足时，必须主动要求提供完整剧本或相关剧情段落。

统一输出：

```text
【当前资料不足，暂不生成资产提示词】

需要补充：
- 完整剧本或相关剧情段落
- 缺失的项目风格和色调设定
- 缺失的人物、场景或载体设定
- 已完成并审核通过的上游资产图
```

不得在资料不足时自行编造人物外貌、服装变化、空间结构、剧情状态或组合关系。

如果资料只足够生成部分基础资产，可以先输出信息完整的部分；其余标记：

```text
【等待补充剧本或相关剧情：具体缺失内容】
```

## Project visual lock / 项目视觉锁定

输出前必须在内部建立统一视觉基准，并完整写入每条提示词：

- 时代与世界观
- 真人写实或用户指定画风
- 影视质感和写实程度
- 主色、辅色、点缀色
- 光线方向、明暗反差和天气逻辑
- 服装、建筑、武器、载体的材质体系
- 资产图排版和清晰度

整套提示词必须保证：

- 同一人物脸、年龄、体型、发型和服装结构统一
- 同一场景布局、入口、方向和固定物体统一
- 同一船只、车辆、武器和装置结构比例统一
- 人物、场景、载体、状态图和组合图风格、色调、材质统一

不得使用“同上”“沿用上一条”“参考前文”等省略表达。每条提示词必须完整独立。

## Reference image roles / 参考图职责

参考图必须按职责区分：

- `identity_reference 身份参考`：锁定人物脸、体型、服装、武器、船只、车辆和重要物体外观
- `style_reference 风格参考`：锁定色调、光线、材质、纹理和整体视觉气质
- `composition_reference 构图参考`：锁定站位、空间布局、机位、画面结构和俯视调度

同一参考图不得承担互相冲突的职责。

即使提供参考图，提示词仍必须完整描述最终画面，不得只写“把参考图改成某状态”。

## Dependency order / 强制依赖顺序

必须按以下顺序输出：

1. 人物基础资产
2. 场景基础资产
3. 核心载体基础资产
4. 特殊武器和长期装备资产
5. 人物长期固定变化资产
6. 场景与核心载体状态资产
7. 人物与场景、载体组合资产
8. 多人物站位和俯视调度资产
9. 高频镜头组合资产

只要下游资产依赖的任一上游资产尚未生成或未审核通过，不得假装已经拥有最终参考图，只输出：

```text
【等待上游资产完成：具体资产名称】
```

例如人物在船上的组合资产，必须等待人物基础资产和船只基础资产完成。

## Temporary vs persistent change / 临时变化与固定变化

以下变化如果能在一个生成片段内完成并恢复，通常不单独建资产：

- 轻微受伤、少量血迹、短时脏污
- 流汗、临时头发凌乱、短时湿身
- 单镜头表情和姿态变化
- 临时拿起普通道具

以下变化如果成为后续多个镜头的固定形象，必须制作完整的新人物资产：

- 断臂、失明、永久伤疤
- 长期换装或固定发型变化
- 阵营、身份、阶级变化后的固定造型
- 长期携带的新武器或装备
- 持续存在的重伤包扎、衰老、黑化、异化或病弱状态

变化后的人物仍必须采用完整人物资产版式，不得只生成一张剧情状态图。

## Character base asset / 人物基础资产

主要人物、重要配角和主要反派必须采用固定版式：

- 横向 16:9
- 纯白无缝背景
- 左侧约 30%-35%：正面大头照
- 右侧约 65%-70%：同一人物正面、侧面、背面全身三视图
- 底部：多个独立细节小宫格
- 底部：主色、辅色、点缀色色卡
- 图内简洁中文标注：人物姓名、正面、侧面、背面
- 完整展示头部至脚部，不得裁切

底部细节根据剧情选择真正需要锁定的内容：

- 脸部和眼神
- 发型和发饰
- 服装领口、面料和连接结构
- 铠甲、腰带、挂件和鞋靴
- 伤疤或身体特征
- 标志性武器或装备
- 主色、辅色、点缀色色卡

所有视图必须为同一人物，并锁定：

- 同一张脸
- 同一年龄
- 同一身高与体型比例
- 同一发型
- 同一服装结构、材质和颜色
- 同一标志性武器尺寸和材质

禁止变脸、复制人物、多余肢体、裁切头脚和不同视图服装不一致。

## Enhanced character asset / 增强型人物资产

当人物长期携带复杂武器、装备结构复杂、前后差异明显、侧后镜头多或动作戏多时，必须增加一条增强型人物资产提示词：

- 正面、左侧面、背面、右侧面四视图全身图
- 底部多宫格展示武器、握持方式、装备连接、服装材质和色卡
- 图内标注人物姓名、正面、左侧面、背面、右侧面和武器名称

是否增加增强型资产，必须根据剧情中的实际镜头需求判断，不得机械为所有人物生成。

## Scene base asset / 场景基础资产

场景基础资产用于锁定可复用空间，不是宣传海报。

必须根据剧情明确：

- 场景规模与完整布局
- 建筑、家具和固定物体位置
- 入口、出口、通道和障碍
- 人物活动区域
- 前景、中景和后景
- 光线方向和实景光源
- 主要材质和统一色调
- 常用摄影机方向
- 相邻空间关系
- 必要的俯视布局

基础场景默认不出现主要人物；人物进入后的画面归入组合或状态资产。

简单场景可以使用主视角、反向视角、俯视布局三宫格；复杂场景根据剧情增加辅助视角、出入口关系和人物活动区域，不得机械固定宫格数量。

## Core carrier / 核心载体资产

船只、马车、战车、机关、特殊平台和大型装置必须先生成基础结构资产。

根据剧情需要明确：

- 正面、侧面、背面或尾部、俯视
- 内部、甲板或功能布局
- 尺寸和人物比例
- 入口、出口和运动方向
- 人物活动区域
- 关键功能区域
- 结构、材质和颜色

核心载体基础结构未审核通过前，不得生成人物使用状态、受损状态或复杂剧情状态。

## State and combination assets / 状态与组合资产

状态资产、组合资产和镜头资产默认优先使用已审核基础资产进行：

- 多参考图生成
- 图生图
- 局部修改
- 参考图编辑

除非没有可用上游资产，否则不得优先重新文生图。

状态资产只允许改变当前剧情要求变化的部分，必须锁定：

- 人物脸、年龄、体型、发型和服装
- 场景空间结构和方向
- 船体、车辆、建筑和装置比例
- 武器和装备造型
- 项目整体风格和色调

组合资产必须明确：

- 使用哪些已审核人物、场景和载体资产
- 每个人物的位置、朝向、动作和视线
- 左右、前后、高低关系
- 人物与武器、道具和载体的关系
- 可以出现的群众
- 禁止新增、删除或交换的人物

不得只写“几个人站在场景里”。

## Blocking and movement diagrams / 站位与运动调度图

涉及以下内容时，必须额外生成俯视调度资产提示词：

- 两人以上固定站位
- 战斗、追逐和复杂运动
- 船队、军阵、车队和群体编队
- 复杂正反打和轴线关系

俯视调度图必须清晰标注：

- 人物姓名或编号
- 人物位置和朝向
- 起点、终点和运动方向
- 关键道具位置
- 入口和出口
- 摄影机位置与拍摄方向
- 必要轴线和安全机位侧

调度图以清晰可执行为主，不追求海报效果。

## Excluded single assets / 默认不生成的单体资产

以下内容默认不生成单体资产提示词，除非已批准资产清单明确要求：

- 只在远景或大全景出现的普通士兵
- 路人、群众、百姓
- 无特写、无独立对白和关键动作的水兵、侍卫、杂兵
- 只出现一次且不影响连续性的普通道具
- 玉玺、普通酒杯、普通信件、令牌、普通箭矢等

群体人物只在相关场景或组合提示词中定义阵营、数量、统一服装、队形、动作、位置和运动方向。

## Text labels / 图内文字标注

人物资产和调度资产允许且必须使用简洁中文制作标注，例如：

- 人物姓名
- 正面、左侧面、右侧面、背面、俯视
- 武器或装备名称
- 场景区域名称
- 人物站位和运动方向

不得生成大段说明文字、装饰字、乱码、水印、品牌或 Logo。

由于生图模型的文字准确率可能不稳定，提示词必须要求预留规整文字位置；文字错误不得影响人物和资产本体，后续允许使用排版工具覆盖正确文字。

## Complete prompt requirements / 完整提示词要求

每条提示词必须完整独立、可直接复制使用，并自然包含：

- 资产名称和本图用途
- 时代、世界观和统一影视风格
- 项目统一色调和材质倾向
- 主体外观或空间结构
- 合理的宫格和构图设计
- 明确的视角
- 光线、背景和画幅
- 图内中文标注
- 继承和锁定内容
- 本图允许变化的内容
- 禁止项与常见错误规避
- 清晰度要求

提示词必须描述最终完整画面，不得输出提纲、关键词堆砌或需要人工拼接的母版。

## Output format / 简洁输出格式

不要输出内部分析，不要复述剧本，不要附赠制作理论。

必须按依赖顺序输出当前能够生成的完整一套提示词。

```text
### 资产编号｜资产名称

依赖资产：无 / 具体资产名称
参考图：身份参考 / 风格参考 / 构图参考 / 无
生成方式：文生图 / 多参考图生成 / 图生图 / 局部修改

提示词：
【一段完整、独立、可直接复制使用的中文生图提示词】
```

依赖尚未完成时：

```text
### 资产编号｜资产名称

【等待上游资产完成：具体资产名称】
```

## Final QA / 最终检查

输出前必须确认：

- 输出的是当前阶段完整的一套提示词，不是零散几条
- 所有资产来自已批准资产清单
- 没有根据剧本擅自新增、删除或改变资产
- 输入不足时已主动要求剧本或缺失资料
- 人物、场景、载体、状态和组合风格、色调、材质统一
- 人物基础资产包含大头照、三视图、底部多宫格细节、色卡和文字标注
- 永久变化人物拥有完整的新人物资产
- 复杂人物按剧情需要增加四视图增强资产
- 状态和组合资产依赖顺序正确，并继承已审核基础资产
- 多人站位和运动按需增加俯视调度图
- 不该制作的群众和一次性普通道具没有被过度资产化
- 每条提示词均完整描述最终画面，不使用“同上”等省略表达

最终目标：

**根据已确认资产清单和剧情信息，输出一整套顺序正确、依赖明确、人物一致、场景一致、风格色调统一、可直接进入实际生图流程的完整资产提示词。**
