# Asset contract / 资产规划与生成契约

## Core principle / 核心原则

资产提取必须先理解完整剧情并推演未来镜头，再决定哪些内容值得制作。

**镜头决定资产，不是剧本名词决定资产。**

资产只服务于：

- 保持主要人物一致性
- 保持核心场景连续性
- 支撑关键动作、站位和镜头调度
- 支撑跨镜头或跨集复用
- 支撑明显且持续的状态变化

不得看到人物、场景或道具名称就机械提取。

## Mandatory reasoning order / 强制判断顺序

1. 理解核心冲突、人物关系、核心场面和视觉重点。
2. 推演主要镜头、景别、机位、人物站位和空间调度。
3. 判断对象是否有清晰识别需求。
4. 判断对象是否需要跨镜头保持一致。
5. 判断对象是否反复使用或存在明显状态变化。
6. 判断制作资产能否减少错误并覆盖足够多镜头。
7. 删除低价值、一次性、可临时生成的内容。
8. 输出资产等级、具体图片、状态、视角、数量、依赖和理由。

不得跳过镜头推演，直接按人物、场景、道具分类罗列。

## Asset value dimensions / 资产价值维度

每个候选内容必须综合判断：

- `recognition_need 识别需求`：是否出现特写、近景或清晰中景，观众是否需要认出它。
- `shot_reuse 镜头复用`：预计服务多少镜头，是否跨场景或跨集反复出现。
- `continuity_need 连续性需求`：外观变化是否会破坏人物、空间或动作连续性。
- `state_change 状态变化`：是否存在明显且需要持续的变化。
- `spatial_value 空间价值`：是否承担人物活动、站位、运动路线和多机位拍摄。
- `production_roi 制作回报`：能否明显减少返工、提高一致性并覆盖多个镜头。

无法证明实际价值的内容，默认不制作。

## Asset levels / 资产等级

### `S` Core asset / 核心资产

承担核心剧情、服务大量镜头、需要多机位、多状态或人物长期活动。

必须建立资产族，按剧情需要包括：

- 基础标准状态
- 必要多视角
- 关键剧情状态
- 人物使用状态
- 多人物组合状态
- 群体或编队状态
- 空间关系图
- 高频镜头参考图

### `A` Important asset / 重要资产

多次清晰出镜、承担重要剧情、需要一定一致性或少量关键状态变化。

制作标准资产，并补充必要状态和常用视角。

### `B` Limited asset / 普通资产

出镜有限，但临时生成容易产生明显偏差。

只制作 1-2 张基础参考图。

### `C` No formal asset / 无需制作

只出现一次、没有近景识别需求、不承担关键动作、不影响连续性、可直接在镜头中生成或可由已有资产组合完成。

C 级内容不得生成正式资产提示词。

## Character asset rules / 人物资产规则

正式人物资产只适用于：

- 主角、重要配角、主要反派
- 多次出现且存在清晰中近景或特写
- 有独立对白、关键动作或重要情绪表演
- 需要跨镜头保持脸、年龄、体型、发型和服装一致

### Standard character sheet / 标准人物资产版式

除非用户明确指定其他格式，必须采用：

- 横向 16:9
- 纯白无缝背景
- 左侧约 30%-35%：正面大头照
- 右侧约 65%-70%：同一人物正面、侧面、背面全身三视图
- 底部：多个独立细节小宫格
- 底部：主色、辅色、点缀色色卡
- 图内简洁标注：人物姓名、正面、侧面、背面
- 全身完整，不裁头、不裁脚

底部细节根据剧情选择：

- 脸部与眼神
- 发型与发饰
- 服装材质、领口、连接结构
- 铠甲、腰带、挂件和鞋靴
- 特殊伤疤或身体特征
- 标志性武器或装备
- 色卡

必须锁定：

- 同一张脸
- 同一年龄
- 同一身高和体型比例
- 同一发型
- 同一服装结构、材质和颜色
- 同一标志性武器尺寸与材质

### Enhanced character sheet / 增强型人物资产

当人物长期携带复杂武器、装备结构复杂、前后差异明显、侧后镜头多或动作戏多时，增加：

- 正面、左侧面、背面、右侧面四视图
- 底部多宫格展示武器、握持方式、装备连接、服装材质和色卡
- 图内标注人物姓名、四个方向和武器名称

不得机械为所有人物生成增强型资产，必须根据剧情镜头需求判断。

### Persistent character changes / 人物长期变化

单个生成片段内可完成并恢复的轻伤、血迹、污渍、表情、姿态和临时持物，通常不单独建资产。

断臂、失明、永久伤疤、长期换装、固定发型变化、身份阵营变化、长期新武器、持续包扎、衰老、黑化或异化等后续持续形象，必须建立完整的新人物资产。

## Crowd and background people / 群体人物规则

以下内容通常不建立单体人物资产：

- 只出现在远景或大全景中的普通士兵
- 路人、百姓、群众、侍卫、水兵、船工
- 无台词、无独立动作、无需观众识别的背景人物

只需要定义：

- 群体类型和阵营
- 大致人数
- 统一服装和色彩
- 年龄与体型范围
- 队形和空间位置
- 动作和运动方向

群体中某人出现清晰中近景、独立对白或关键动作时，才升级为正式人物资产。

## Scene asset rules / 场景资产规则

正式场景资产适用于：

- 承担大量剧情
- 人物反复活动
- 需要复杂站位和空间调度
- 需要正反打、运动镜头或多方向拍摄
- 场景结构错误会破坏连续性
- 存在关键环境变化

必须明确：

- 空间规模
- 固定建筑、家具和物体布局
- 入口、出口、通道和障碍
- 人物活动区域
- 前景、中景和后景
- 常用机位和摄影机移动路线
- 光线方向和实景光源
- 主要材质和统一色调
- 相邻空间关系

核心场景不能只制作一张无人空景。必须根据剧情判断是否还需要：

- 标准空场景
- 主要人物进入后的状态
- 多人物站位状态
- 战斗前、中、后状态
- 白天、夜晚、浓雾、火光等关键环境状态
- 人物主观视角
- 高频正反打方向
- 俯视布局和调度图

基础场景默认不出现主要人物；人物进入后的画面属于组合或状态资产。

## Core carrier rules / 核心载体规则

不得因为某对象形式上属于“道具”，就降低其资产等级。

船只、马车、战车、机关、特殊建筑、关键武器、法器、机械装置和人物长期活动的平台，如果承担大量镜头，应作为核心载体建立资产族。

核心载体基础资产按剧情需要包括：

- 正面、侧面、背面或尾部、俯视
- 内部、甲板或功能布局
- 尺寸和人物比例
- 入口、出口和运动方向
- 人物活动区域
- 关键功能区域
- 结构、材质和颜色

核心载体基础结构未审核通过前，不得生成人物使用状态或复杂剧情状态。

例如草船资产族可包括：

- 空船基础结构
- 正面、侧面、俯视
- 甲板布局
- 人物就位状态
- 击鼓和划桨位置
- 浓雾航行
- 接近曹营
- 箭雨中
- 船舷逐渐插箭
- 满箭返航
- 多船编队
- 靠岸
- 船头看船尾、船尾看船头和船舷近景

## Prop rules / 普通道具规则

普通酒杯、信件、令牌、玉玺、箭矢、桌椅和碗筷等，只出现一次且不影响连续性时，不建立正式资产，直接写入镜头提示词。

只有满足以下条件时才升级：

- 外观高度独特
- 多次清晰出镜
- 参与关键动作
- 是重要视觉线索
- 需要多个镜头保持完全一致
- 临时生成极易出错

剧情意义重要，不等于必须建立资产。

## Asset family / 资产族

同一对象存在多个剧情阶段、状态或高频组合时，必须归入同一资产族：

- `base_asset 基础资产`：稳定外观、结构、比例和材质
- `state_asset 状态资产`：关键剧情阶段的明显变化
- `combination_asset 组合资产`：人物、场景、载体之间反复使用的固定关系
- `shot_asset 镜头资产`：高频人物、站位、场景和机位组合参考

镜头资产只在能服务多个镜头时制作，不得为每个单独镜头建立一份资产。

## Dependency and generation order / 依赖与生成顺序

必须按以下顺序：

1. 人物基础资产
2. 场景基础资产
3. 核心载体基础资产
4. 特殊武器和长期装备
5. 人物长期固定变化资产
6. 场景与载体状态资产
7. 人物与场景、载体组合资产
8. 多人物站位和俯视调度资产
9. 高频镜头组合资产

上游未审核通过，下游不得假装可以最终生成。

状态、组合和镜头资产默认优先基于已审核基础资产做多参考图、图生图、编辑或局部修改，不得无必要从零文生图。

## Stable IDs / 稳定编号

- `C01-C99` characters / 人物
- `S01-S99` scenes / 场景
- `P01-P99` props and carriers / 道具与载体
- `F01-F99` asset families / 资产族
- `G01-G99` crowd groups / 群体规则
- `A01-A99` combination or shot assets / 组合或镜头资产

## Prompt and label rules / 提示词与标注规则

每个提示词必须完整、独立、可直接用于生图，包含：

- 项目统一风格与色调
- 主体外观或空间结构
- 材质、颜色、构图、背景、光线和画幅
- 继承与锁定内容
- 当前允许变化的内容
- 禁止项

允许并要求必要的制作标注：

- 人物姓名
- 正面、左侧面、右侧面、背面、俯视
- 武器或装备名称
- 场景区域名称
- 人物站位和运动方向

禁止：

- 装饰性大段文字
- 乱码、水印、Logo、UI框和装饰边框
- 重复肢体、重复武器和多余人物
- 变脸、服装不一致和结构漂移
- 裁切人物头脚
- 妨碍参考使用的透视畸变和运动模糊
- 海报式构图
- 遮挡空间布局的烟火、粒子和过度氛围

## Required extraction output / 资产提取强制输出

资产提取阶段必须输出以下六部分：

### 1. `story_and_shot_understanding 剧情与镜头理解`

- 核心冲突
- 主要人物
- 核心叙事空间
- 重要视觉场面
- 高频镜头类型
- 资产制作重点

不得长篇复述剧情。

### 2. `required_assets 必须制作的资产`

每项包含：

- 资产名称与稳定 ID
- 类型与等级 S/A/B
- 剧情作用
- 预计服务镜头
- 识别景别
- 一致性需求
- 是否建立资产族
- 多状态、多视角和人物组合需求
- 建议图片数量
- 必须生成的每张具体图片
- 依赖资产
- 制作理由

### 3. `core_asset_families 核心资产族`

- 资产族名称与 ID
- 基础状态
- 关键剧情状态
- 人物使用状态
- 组合站位
- 常用机位
- 群体或编队状态
- 建议生成顺序

### 4. `crowd_handling 群体元素处理`

- 群体名称与 ID
- 使用场景和主要景别
- 是否需要单体资产：默认否
- 服装规则
- 数量和队形
- 动作规则
- 镜头中如何生成

### 5. `excluded_assets 无需制作的内容`

- 内容名称
- 不制作原因
- 镜头中如何处理

必须主动排除一次性物品、路人、群众和普通道具。

### 6. `production_order 最终制作顺序`

- 第一批：拍摄前必须完成的核心资产
- 第二批：进入分镜后补充的重要资产
- 第三批：根据实际镜头再制作的资产
- 无需制作：直接在镜头生成时处理

## QA gate / 输出前自检

- 是否先推演镜头，再决定资产
- 是否遗漏核心空间和核心载体
- 是否只做空场景，遗漏人物使用状态
- 是否遗漏关键状态变化和长期固定变化
- 是否遗漏多人物站位、群体编队和运动方向
- 是否为远景士兵、路人或群众错误制作单体资产
- 是否把一次性普通道具过度资产化
- 是否存在重复资产
- 是否明确列出每张需要生成的图片
- 是否按依赖顺序生成
- 是否保证人物、场景、载体、风格、色调和材质统一

最终目标：

**用最少、最准确、最可复用的资产，支撑最多的有效镜头。**
